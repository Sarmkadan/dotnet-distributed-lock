# Contributing to dotnet-distributed-lock

First off, thank you for considering contributing to dotnet-distributed-lock! It's people like you that make it such a great tool.

## Development Requirements

- **.NET SDK**: .NET 10.0 SDK is required for development and testing.

## How to Contribute

### 1. Fork and Clone

- Fork the repository on GitHub.
- Clone your fork locally: `git clone https://github.com/<your-username>/dotnet-distributed-lock.git`

### 2. Create a Branch

Create a descriptive branch for your changes:
`git checkout -b feature/your-feature-name` or `git checkout -b fix/your-bug-fix`

### 3. Make Changes

- Make your code changes.
- Ensure you follow the project's **Code Style**:
  - Follow existing conventions.
  - Include XML docs for public APIs.
  - **IMPORTANT**: Keep all existing author headers intact. Do not remove them.

### 4. Run Tests

Ensure all tests pass before submitting your pull request. Run the tests using:
`dotnet test`

### 5. Submit a Pull Request

- Push your branch to your fork: `git push origin your-branch-name`
- Open a Pull Request on the main repository.
- Provide a clear and detailed description of the changes you've made.

## Reporting Issues

We use GitHub Issues to track bugs and feature requests. 

When reporting a bug, please include:
- A clear and descriptive title.
- Steps to reproduce the issue.
- Expected behavior vs. actual behavior.
- Version of the library and environment details.
